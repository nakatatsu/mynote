# タスク1 採点用チェックリスト（10点満点）

※ Issueには記載しない。採点者のみが使用する。

各項目を ○（1点）/ ×（0点）で判定する。

| # | 判定基準 | 得点 |
|---|---------|------|
| 1 | `root / modules / envs`（またはそれに準ずる）のディレクトリ分離がある | /1 |
| 2 | ALB / ECS / RDS をそれぞれモジュール分割し、責務が明確に記述されている | /1 |
| 3 | ネットワーク（VPC / Subnet / RouteTable / NAT）を独立モジュールとして設計している | /1 |
| 4 | セキュリティ（SG / IAM / KMS / Secrets Manager 等）を扱う方針に言及している | /1 |
| 5 | stateバックエンド（S3 + DynamoDB ロック等）に具体的に言及している | /1 |
| 6 | workspace または env ディレクトリ戦略が一貫して説明されている | /1 |
| 7 | variables に型・description が付与された定義例がある | /1 |
| 8 | outputs が依存関係に有用な値（ALB DNS、ECS cluster ARN 等）を返している | /1 |
| 9 | remote state 参照や依存の扱いが破綻していない（矛盾がない） | /1 |
| 10 | マルチ環境（dev / staging / prod）のディレクトリ戦略に具体的な言及がある | /1 |

**合計: /10**
