# Benchmark Spec

## 目的

Claude Code と Codex を毎月比較し、業務での使い分け判断の材料にする。
結果は GitHub リポジトリに公開する。

## 評価タスク

以下の3カテゴリを毎月固定タスクで実施する。

- **設計**: 仕様から設計書・構成案を出力させる
- **コード生成**: 要件からコードを生成させる
- **レビュー**: 既存コードのレビューコメントを出力させる

各タスクは独立したセッションで実行する（前のタスクの文脈を持ち越さない）。

## 実行環境

DevContainer 上で実施。

## 自動化範囲

- GHA が月次で Issue を作成する（実施リマインダー＋チェックリスト）
- そのIssueを見てCodexとClaude Codeがそれぞれ実行

## レポート

`benchmark/reports/YYYY-MM/result.md` として本リポジトリに毎月コミット。

### 構成

- タスク別スコアサマリー（★5段階、ざっくり）
- タスクごとに：お題 / Claude Code の成果物 / Codex の成果物

Claude CodeとCodexがたがいにそれぞれ、どちらの成果物かは知らせず、双方に両方とも評価させる

## 各タスクの定義

Issueに記載する

## 成果物の収集方法

`benchmark/reports/YYYY-MM/artifacts/` 以下に保存
